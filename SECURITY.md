# Security Policy

В этом проекте используются только учебные данные и учебный код, однако базовые правила безопасной разработки всё равно соблюдаются.

Если вы заметили потенциальную уязвимость или некорректное обращение с данными, пожалуйста, сообщите об этом через лс в тг:

Telegram: https://t.me/Tengir

Пожалуйста, укажите:
- что именно вызывает подозрение,
- при каких условиях это проявляется,
- желательно — короткое описание, как воспроизвести.

Публичные issue с описанием уязвимостей не создаются.

Общие требования:
- не хранить ключи, токены и другие секреты в репозитории;
- не использовать реальные персональные данные;
- при необходимости работы с конфигурацией использовать `.env` (пример — в `.env.example`).

Этот документ носит учебный характер и используется в рамках курса.
