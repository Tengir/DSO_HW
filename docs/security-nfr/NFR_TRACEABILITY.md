## Описание User Stories

**US-01:** Создание колод
Пользователь может создавать новые колоды с названием, описанием и языками. Система проверяет корректность данных перед сохранением.

**US-02:** Просмотр списка колод
Пользователь видит все свои колоды в списке. Запрос выполняется быстро, даже если колод много.

**US-03:** Вход в аккаунт
Пользователь входит в систему с логином и паролем. Пароли хранятся безопасно, система защищена от брутфорса.

**US-04:** Получение ошибок
Пользователь получает понятные ошибки в едином формате при любых проблемах (валидация, доступ, не найдено).

**US-05:** Защита данных
Пользователь видит только свои колоды и карточки. Попытка доступа к чужим ресурсам блокируется.

**US-06:** Быстрый отклик API
Пользователь получает ответы от API быстро, без долгих задержек при работе с карточками.

## Легенда приоритетов

- **Must** — обязательное требование, без него нельзя продолжать
- **Should** — желательное, можно отложить, но лучше сделать

## Легенда Milestone

- **P03** — текущая практика (NFR, архитектура)
- **P04** — аутентификация и авторизация
- **P05** — производительность и оптимизация
- **P06** — логирование и мониторинг
- **P07** — надёжность и доступность
